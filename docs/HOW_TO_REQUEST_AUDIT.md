# How to Request a Smart Contract Audit

Smart contract security is critical. This guide walks you through requesting a professional smart contract audit from Aetheron Sentinel—from initial contact through remediation.

## Overview

A smart contract audit is a comprehensive security review of your smart contract code, logic, and architecture. Our audit includes:

- **Automated scanning** - Detecting known vulnerability patterns and code anti-patterns.
- **Manual code review** - Expert auditors analyzing business logic, state management, and attack vectors.
- **Architecture assessment** - Evaluating contract interactions, access controls, and upgradability.
- **Risk report** - Detailed findings with severity levels and remediation recommendations.
- **Remediation support** - Guidance on fixing identified issues.

**Typical timeline:** 2–4 weeks (standard audit), 1–2 weeks (expedited).  
**Pricing:** Professional tier ($99/month), Enterprise (custom quotes).

## Step 2: Provide Scope & Details

Our team will request the following information within 24 hours:

### Contract Files
- Upload or link to your smart contract source code (GitHub, .zip, or Etherscan).
- Supported languages: **Solidity, Rust (Anchor), Vyper, Python (Cairo)**.
- Include: ABI, deployment scripts, and test suite (if available).

### Blockchain & Chain
- Which blockchain? (Ethereum, Polygon, Arbitrum, Base, Solana, Cosmos, etc.)
- Mainnet, testnet, or both?

### Complexity & Scope
- How many contracts? (e.g., 1–5 contracts, 5–15, 15+)
- Estimated lines of code (LOC)?
- Does it interact with external protocols? (DEX, lending, bridges, oracles)

### Timeline Preference
- Standard audit (2–4 weeks): $5,000–$15,000.
- Expedited audit (1–2 weeks): +50% premium.
- Enterprise/Retainer: Custom pricing.

### Specific Concerns
- Are there areas of concern? (cross-contract interactions, oracle dependencies, access control)
- Previous audits or security reviews?

---

## Step 3: Receive Audit Timeline & Quote

Within **3–5 business days**, we'll send you:

1. **Audit Scope Document** - Lines of code, complexity assessment, timeline.
2. **Pricing Estimate** - Breakdown of costs, delivery timeline, milestones.
3. **SLA Agreement** - Confidentiality terms, liability limitations, remediation support included.
4. **Payment Terms** - 50% deposit to start, 50% on delivery (or agreed installment plan).

**Example timeline:**
- Week 1: Initial code review + vulnerability scanning.
- Week 2: Deep manual analysis + documentation.
- Week 3: Findings compilation + severity assessment.
- Week 4: Final report delivery + remediation recommendations.

---

## Step 4: Audit Execution & Reporting

Once you approve the scope and pricing:

1. **Payment:** Send 50% deposit to initiate audit.
2. **Code Access:** Grant access to GitHub repository or share code securely.
3. **Kick-off Call:** Optional 30-minute call to discuss scope, concerns, and timeline.
4. **Audit Phase:** Our team audits your contracts over 2–4 weeks.
5. **Final Report:** Receive comprehensive audit report including:
   - Executive summary
   - Detailed findings (critical, high, medium, low severity)
   - Code snippets & attack descriptions
   - Remediation recommendations
   - Risk scorecard

---

## Step 5: Remediation & Follow-Up

After receiving the audit report:

### Review Findings
- Have your team review audit findings carefully.
- Ask questions—our auditors are available for clarification calls.

### Implement Fixes
- Fix identified issues following our remediation recommendations.
- Conduct unit tests and integration testing for each fix.

### Re-Audit (Optional)
- Request a **targeted re-audit** ($1,000–$3,000) to verify fixes for critical/high findings.
- Recommended before mainnet deployment.

### Deploy with Confidence
- Publish your audit report (optional but recommended for trust).
- Deploy to testnet first; monitor for anomalies.
- Deploy to mainnet only after thorough testing.

## Important Notes

### What We Audit
✓ Smart contract code and logic  
✓ State management and access controls  
✓ External protocol interactions  
✓ Gas optimization opportunities  

### What We Don't Audit
✗ Off-chain backend systems  
✗ Dapp frontend code (React, TypeScript)  
✗ DevOps or deployment infrastructure  
✗ Regulatory compliance (consult a lawyer)  

### Risk Disclaimer
See [DISCLAIMERS.md](DISCLAIMERS.md) for important audit limitations. **No audit is 100% foolproof.** Even after a professional audit, vulnerabilities may remain undetected. We strongly recommend third-party audits (OpenZeppelin, Trail of Bits, Quantstamp) for high-stakes contracts.
